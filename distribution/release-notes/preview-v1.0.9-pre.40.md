## What's changed

- Replaced the accessibility consent dialog with a dedicated full-page disclosure.
- Removed the "Skip for now" action from onboarding so the introduction continues to the permission decision page.
- Added clear feedback when leaving the accessibility disclosure with the system Back action.

## Accessibility consent

- Accessibility settings open only after selecting "Agree: use accessibility service."
- Selecting the decline option returns to the app without enabling the service.
- Pressing Back is treated as a decline and confirms that the accessibility service was not enabled.
