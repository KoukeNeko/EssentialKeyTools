## What changed

- Key Test now previews detected key events and classified gestures without executing the action
  mapped to that gesture.
- A gesture that begins on Key Test remains suppressed even if its classification finishes shortly
  after leaving the screen, preventing delayed single-press actions from firing unexpectedly.
- GitHub releases now include the signed APK's SHA-256 digest and its VirusTotal and Koodous
  submission status, with report links when available.
- Re-running the release workflow for an existing tag now refreshes both its APK/AAB assets and its
  release notes.

## Release artifact security

The release workflow checks whether the signed APK already exists on VirusTotal and Koodous before
uploading it. Provider errors, rate limits, or an unavailable optional credential are reported in
GitHub Actions without blocking the GitHub or Google Play release.

Only the signed APK intended for public release is submitted. Hardware key events, saved mappings,
settings, and other device data are not included or transmitted by this workflow.

- The security-submission script passed ShellCheck and a no-credential dry run.
- The GitHub Actions workflow passed YAML parsing and whitespace validation.
