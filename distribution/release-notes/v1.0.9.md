## What's changed

- Added a guided three-step onboarding flow with language selection, an introduction, and a clear
  accessibility decision page.
- Replaced the accessibility consent dialog with a dedicated full-page disclosure using clearer,
  less technical explanations.
- Added a manual update check that uses Google Play for Play installations and GitHub Releases for
  sideloaded or preview installations.
- Prevented Key Test from launching the action assigned to a detected gesture.
- Removed the legacy Shizuku and ADB setup paths.

## Accessibility and privacy

- Android accessibility settings open only after the user explicitly chooses to use the
  accessibility service.
- Choosing not to use the service, or leaving the disclosure with Back, does not enable it.
- Essential Key events are processed immediately on the device and are not stored, uploaded, or
  shared. Only the confirmed Essential Key identifier is saved locally.
- Screenshot and lock-screen actions run only when the user explicitly assigns them to a shortcut.

## Update checks

- Update checks run only when requested by the user and never download or install an APK directly.
- Stable builds do not offer preview releases, and preview builds can be installed beside the
  production app for testing.

## Validation

- Android lint, JVM unit tests, and debug APK assembly pass the project quality gate.
- Release APKs are submitted to VirusTotal and Koodous by CI when the optional credentials are
  available; provider availability does not block publication.
