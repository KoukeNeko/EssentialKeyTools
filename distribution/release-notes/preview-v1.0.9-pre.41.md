## What's changed

- Rewrote the accessibility consent text in clearer, less technical language.
- Updated both onboarding and the full-page accessibility disclosure with consistent explanations.
- Simplified the accessibility service description shown by Android settings.

## Clearer privacy details

- Explains that the app receives which hardware key was pressed or released and when it happened.
- Clarifies that key events are processed immediately on the device and are not stored, uploaded, or shared.
- States that only the confirmed Essential Key identifier is stored on the device.
- Explains that screenshot or lock-screen actions run only when the user assigned them to a shortcut.
