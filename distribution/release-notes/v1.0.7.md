## What changed

- Removed all Shizuku integration, including its API/provider dependencies, Binder provider,
  permission flow, privileged shell runner, and one-tap service controls.
- Removed the ADB command path from the single-press unlock wizard.
- Accessibility-service setup now uses Android's Accessibility Settings after the existing consent
  disclosure.
- Single-press unlock now uses only the package App Info pages for manual disable or restore, while
  keeping live per-package status.
- Removed outdated screenshots and synchronized the README, privacy policy, translations, and Play
  Store description with the supported flows.

## Privacy and permissions

The app no longer includes a privileged Binder provider or communicates with a separate privileged
service. Its declared capabilities are limited to the accessibility service, package visibility
needed by the action picker and unlock status, and network access for the public GitHub contributor
list.

## Validation

- `./scripts/verify.sh`: Android lint, JVM unit tests, and debug APK assembly passed.
- Release Kotlin compilation and R8 minification passed.
- Source, runtime dependencies, merged manifests, and the debug APK contain no Shizuku integration.
- The debug APK contains no ADB unlock commands or command-copying flow.
